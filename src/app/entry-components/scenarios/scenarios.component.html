Reformatting (or removing) previously-committed log messages that violate pre-commit auditing

 When would I need to do this?
  • Say that your team or organization is checking individual commit messages for a certain format string or identifier from an external application in a pre-commit hook.
  • If you have multiple commits with improperly formatted messages, you will have to rewrite or eliminate those problematic commit messages in order to integrate with the monitored remote repository.
What are some ways to fix this problem?
  • Squash the commits into one commit with a compliant log message.
  • Amend the improperly formatted log messages so they are compliant.
